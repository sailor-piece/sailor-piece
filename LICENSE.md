## 🐛 Common Issues and Fixes

### ❌ GUI doesn't appear after execution
**Fix:** Press `RightShift` or `Insert` to manually toggle the GUI. If still not visible, disable Discord overlay, GeForce Experience overlay, or active ReShade filters — then re-inject and execute.

### ❌ Auto Boss not teleporting to boss locations
**Fix:** This usually occurs after a game update changes boss spawn coordinates. Update to the latest script version — Auto Boss routing is the first module patched after every Sailor Piece world update.

### ❌ Fruit Sniper not collecting fruits
**Fix:** The Fruit Sniper monitors server-wide spawn events. If fruits aren't being collected, the spawn event name may have changed in a recent game patch. Update to the latest version and toggle the module off and on to reinitialize the event listener.

### ❌ Kill Aura not hitting NPCs
**Fix:** Confirm that Kill Aura is activated after the game session fully loads (not during the loading screen). If still not working, toggle it off and on to reset the targeting loop. Reduce the radius if performance lag is causing hit registration delays.

### ❌ Script not working on Android
**Fix:** Ensure you're using **Delta v2.3+** or **Arceus X v2.0+** and that Roblox was launched through the executor's built-in launcher — not independently. Wait for the game to fully load into the server before executing.

### ❌ God Mode deactivating unexpectedly
**Fix:** Some Sailor Piece boss attacks use scripted forced-damage that bypasses standard God Mode hooks. Update to the latest version — God Mode is regularly patched to cover new boss AoE mechanics. The settings panel also includes a "Hard God Mode" toggle for these cases.

### ❌ "attempt to index nil value" Lua error
**Fix:** A recent Sailor Piece update broke a variable reference. Check the issues tab of this repository for a hotfix announcement, or wait for the next scheduled patch release.
